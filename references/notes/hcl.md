# Hard Competitive Learning (HCL)

## 概要

Hard Competitive Learning (HCL) は、最も単純な競合学習アルゴリズムである。
各入力に対して最近傍ノード（勝者）のみを更新する "winner-take-all" 方式を採用する。

## 元論文

- Rumelhart, D. E., & Zipser, D. (1985). "Feature Discovery by Competitive Learning"
  Cognitive Science, 9(1), 75-112.

## アルゴリズム

1. N 個のノードをランダムに初期化
2. 入力信号 ξ に対して:
   - 最近傍ノード（BMU: Best Matching Unit）を見つける
   - 勝者のみを入力方向に移動: `w_bmu += ε * (ξ - w_bmu)`
3. 学習率 ε を徐々に減衰（通常は指数減衰）

## パラメータ

| パラメータ | 説明 | 典型値 |
|-----------|------|--------|
| n_nodes | ノード数（固定） | 50-100 |
| learning_rate_initial | 初期学習率 | 0.3-0.5 |
| learning_rate_final | 最終学習率 | 0.01-0.05 |

## demogng.de 実装

demogng.de では HCL は以下のように実装:

```javascript
// 勝者のみ更新
bmu.pos.add(Vec.sub(signalPos, bmu.pos).scale(epsilon));
```

学習率の減衰:
```javascript
epsilon = initEpsilon * Math.pow(finalEpsilon / initEpsilon, t);
```

## 特徴

### 利点
- **シンプル**: 実装が最も簡単
- **高速**: 勝者のみ更新するため計算コストが低い
- **ベースライン**: 他のアルゴリズムとの比較基準になる

### 欠点
- **トポロジー学習なし**: ノード間の接続関係を学習しない
- **dead units 問題**: 一部のノードが使われなくなる可能性
- **局所解**: 初期配置に依存しやすい
- **追従性が低い**: 勝者のみの更新で適応が遅い

## 他のアルゴリズムとの比較

| アルゴリズム | 更新範囲 | トポロジー | ノード数 |
|-------------|---------|-----------|---------|
| HCL | 勝者のみ | なし | 固定 |
| Neural Gas | ランク順で全ノード | なし | 固定 |
| SOM | 近傍関数で減衰 | グリッド固定 | 固定 |
| GNG | 勝者+直接隣接 | 自動学習 | 可変 |

## 使用場面

- ベクトル量子化の簡単な実装
- 他のアルゴリズムのベースライン比較
- 計算リソースが限られた環境
- 高速なオンライン学習が必要な場合（精度は犠牲）

## 注意点

HCL は追従性（トラッキング能力）が低いため、非定常分布には不向き。
学習率を高く保てば多少の追従は可能だが、収束性とのトレードオフがある。
